## 二、核心目录职责说明

| 目录 | 职责 | 说明 |
|------|------|------|
| `app/` | 应用路由与页面 | Next.js App Router 入口；每模块独立子路由；API 路由集中于 `app/api/` |
| `app/api/` | HTTP API 端点 | 留言、联系表单、简历下载等后端逻辑 |
| `components/` | 可复用 UI 组件 | 按业务域划分（resume/portfolio/links/guestbook）；`ui/` 为通用基础组件 |
| `lib/` | 业务逻辑与基础设施 | 数据库、邮件、存储、Server Actions、工具函数 |
| `content/` | 静态内容源 | JSON/Markdown 驱动的履历、作品集、链接配置；支持后期迁移至 CMS；`lib/content/*.ts` 加载失败时需区分 `ENOENT`（文件不存在）与 `SyntaxError`（解析错误），禁止空 catch 吞掉所有异常，至少 `console.error` 输出错误类型 |
| `public/` | 静态资源 | 图片、PDF、favicon 等 |
| `drizzle/` | 数据库迁移 | 版本化 schema 变更 |
| `docs/` | 架构与接口文档 | 唯一真相源，开发必读 |

---

## 三、变更管理

- **新增模块**：在 `app/` 下新增路由目录，在 `components/` 下新增对应组件目录。
- **API 变更**：需同步更新 `docs/04-API接口文档.md`。
- **数据库变更**：需同步更新 `docs/03-数据库设计.md` 并生成 migration。
