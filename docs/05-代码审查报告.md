## 二、问题清单

### 🔴 严重问题（必须修复）

| 文件 | 行号 | 问题描述 | 修复建议 |
|------|------|----------|----------|
| `app/api/guestbook/[id]/route.ts` | 20-36 | **DELETE 接口无鉴权**：任何人可删除任意留言，与 API 文档「需管理员 token」不符 | 1) 临时方案：添加简单 API Key 校验（`Authorization: Bearer <ADMIN_TOKEN>`）<br>2) 生产方案：接入 NextAuth 或类似鉴权，校验管理员身份 |
| `lib/utils/performance.ts` | 52-64 | **rafThrottle 实现与类型不符**：返回函数内部再次返回取消函数，类型声明为 `() => void` 错误；且非典型节流实现 | 修正为标准 rAF 节流：`export function rafThrottle(fn: () => void): () => void { let rafId: number \| null = null; return () => { if (rafId != null) return; rafId = requestAnimationFrame(() => { fn(); rafId = null; }); }; }` |
| `lib/rate-limit.ts` | 14 | **内存存储无限增长**：`Map<ip, Map<key, entry>>` 无清理机制，长时间运行会 OOM | 增加定时清理或 LRU：定期删除超窗外的条目，或使用 `TemporalStore` 限制最大条目数 |
| `lib/db/schema.ts` + `app/api/resume/download/route.ts` | - | **resume_download_log 表未使用**：架构文档要求「可写入用于统计与限流」，当前未写入 | 在 `app/api/resume/download/route.ts` 成功返回 URL 后，调用 `db.insert(resumeDownloadLog).values({ ipHash: hash(ip), userAgent })` 写入日志；需新增 `lib/db/queries/resume.ts` 封装 |
