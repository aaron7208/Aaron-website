# 上线检查清单

> 本地预览通过 ≠ 上线无忧。上线前请逐项确认。

---

## 一、Vercel 部署（推荐）

### 1. 上线前必做

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 创建 Turso 数据库 | 访问 [turso.tech](https://turso.tech)，创建数据库，获取 `DATABASE_URL` 与 `TURSO_AUTH_TOKEN` |
| ② | **在 Turso 上建表** | 本地执行：`DATABASE_URL="libsql://xxx.turso.io" TURSO_AUTH_TOKEN="xxx" npm run db:init`（或 `npm run db:push`） |
| ③ | 配置 Vercel 环境变量 | `DATABASE_URL`、`TURSO_AUTH_TOKEN` 必填；`RESEND_API_KEY`、`SITE_OWNER_EMAIL` 按需 |
| ④ | 配置 `ADMIN_API_KEY` | 若需使用 DELETE 留言接口，必须配置 |
| ⑤ | 提交代码并部署 | 连接 Git 仓库，Vercel 自动构建 |

### 2. 关键区别（本地 vs 上线）

| 项目 | 本地开发 | Vercel 生产 |
|------|----------|-------------|
| 数据库 | `file:./local.db`（本地文件） | **必须** Turso（`libsql://xxx.turso.io`） |
| 限流 | 内存 Map，单实例有效 | ⚠️ Serverless 多实例，内存限流失效；建议接入 Upstash Redis |
| 建表 | 本地 `npm run db:init` | **上线前** 需用生产 `DATABASE_URL` 执行一次建表 |

### 3. 常见失败原因

- **留言板/联系表单报错**：Turso 未建表 → 执行步骤 ②
- **联系表单无邮件**：未配置 `RESEND_API_KEY`、`SITE_OWNER_EMAIL`
- **构建失败**：检查 Node 版本（建议 18+）、依赖是否完整

### 4. 代理与限流 IP（安全）

- 限流依赖「客户端 IP」识别，Vercel 等平台会设置 `X-Forwarded-For`，应用会取首段作为 IP。
- **请勿在应用前再挂可被用户控制的代理**，否则恶意用户可伪造 `X-Forwarded-For` 绕过或干扰限流。若使用自建 Nginx/云 WAF，请确认仅由可信代理设置该头。详见 `docs/26-安全审查报告.md`。

## 二、Docker 自托管部署

### 1. 上线前必做

| 步骤 | 操作 |
|------|------|
| ① | 修正 `docker-compose.yml` 中 `DATABASE_URL` 为容器内路径（见下） |
| ② | 首次启动后进入容器执行 `npm run db:init`，或挂载已初始化的 `local.db` |
| ③ | 配置 `ADMIN_API_KEY`（若需 DELETE 接口） |

### 2. Docker 数据库路径修正

当前 `DATABASE_URL=file:./data/local.db` 与 volume `./data:/app/data` 配合时，应改为：

```yaml
environment:
  - DATABASE_URL=file:/app/data/local.db
```

### 3. 首次建表

容器启动后，`/app/data` 为空，需执行建表。可选方式：

- 启动后 `docker exec -it <container> node scripts/db-init.mjs`（需将脚本打入镜像）
- 或在 Dockerfile 的 CMD 前增加建表步骤（不推荐，每次重启都跑）
- 或挂载已包含 `local.db` 的目录

## 三、通用检查项

| 检查项 | 说明 |
|--------|------|
| `public/resume.pdf` | 简历下载需此文件，否则 404 |
| 作品集缩略图 | 若使用外部图床，需在 `next.config.js` 的 `remotePatterns` 中配置域名 |
| 限流 | 生产环境建议接入 Upstash Redis，替代内存限流 |
| 构建验证 | 本地执行 `npm run build` 确保无报错 |
| 依赖安全 | 建议 CI 中增加 `npm audit --audit-level=high`；定期执行并升级有漏洞依赖，见 `docs/26-安全审查报告.md` |

---

## 四、结论

**当前成果在完成上述配置后可顺利上线**，但需注意：

1. **Turso 建表**：上线前务必用生产 `DATABASE_URL` 执行一次 `db:init` 或 `db:push`
2. **限流**：Vercel Serverless 下内存限流不可靠，高流量场景需接入 Upstash
3. **Docker**：需修正数据库路径并处理首次建表流程
