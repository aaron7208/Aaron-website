## 二、drizzle.config.ts 配置修正（v0.20 API 变更）

**问题**：`dialect: "sqlite"` + `dbCredentials.url` 报错「url 不在类型中」。

**原因**：drizzle-kit v0.20+ 使用 `driver` 替代 `dialect`，且不同 driver 对应不同 `dbCredentials` 结构。

**修复**：
- 本地开发（无 TURSO_AUTH_TOKEN）：`driver: "libsql"`，`dbCredentials: { url }`
- 生产/Turso：`driver: "turso"`，`dbCredentials: { url, authToken }`

## 三、依赖安装前置条件

**以下错误在 `node_modules` 缺失时会出现，需先执行依赖安装：**

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| 找不到模块 "drizzle-kit" | devDependency 未安装 | `npm install` |
| 找不到模块 "tailwindcss" | dependency 未安装 | `npm install` |
| 找不到 "node" 的类型定义文件 | `@types/node` 未安装 | `npm install` |
| 找不到名称 "process" | 同上 | `npm install` |

```bash
# 在项目根目录执行
npm install
# 或
pnpm install
```

## 五、验证步骤

安装依赖后，执行以下命令验证类型无报错：

```bash
npm install
npx tsc --noEmit
# 或通过 Next.js 构建
npm run build
```

## 六、若仍有类型错误

1. **确认 `node_modules` 已存在**：`ls node_modules` 或 `dir node_modules`
2. **确认 `@types/node` 已安装**：`ls node_modules/@types/node`
3. **重启 IDE / TypeScript 服务**：有时需要重新加载项目
4. **检查 Node 版本**：建议 Node 18+，`node -v`
