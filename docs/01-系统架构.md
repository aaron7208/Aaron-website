## 一、需求概览

| 模块 | 功能描述 |
|------|----------|
| 个人履历 | 展示教育、工作经历、技能树 |
| Vibe-Coding 作品集 | 项目卡片、技术栈标签、链接/预览 |

## 二、技术栈选型

### 2.1 前端技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **框架** | Next.js 14 (App Router) | 支持 SSG/ISR 静态生成，首屏快；同时支持 React 丰富交互；与 Vercel 天然适配 |
| **语言** | TypeScript | 企业级类型安全，可维护性高 |
| **样式** | Tailwind CSS | 原子化 CSS，便于快速实现视觉设计 |
| **动画/交互** | Framer Motion | 声明式动画，手势、布局动画能力强 |
| **3D/沉浸** | Three.js + React Three Fiber | 可选：用于 3D 背景、粒子、空间感（满足「视觉酷」） |
| **滚动体验** | Lenis | 平滑滚动，提升交互质感 |
| **图标** | Lucide React | 轻量、一致、可定制 |

### 2.2 后端技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **API** | Next.js API Routes / Server Actions | 无额外服务，与前端同仓，部署简单 |
| **ORM** | Drizzle ORM | 轻量、类型安全，与 SQLite/Postgres 适配好 |
| **数据库** | SQLite (开发) / Turso (生产) | 个人规模足够；Turso 为边缘 SQLite，低延迟、免费额度友好 |
| **文件存储** | Vercel Blob 或 Cloudflare R2 | 简历 PDF、作品集资源上传 |
| **邮件** | Resend 或 Nodemailer + SMTP | 联系表单、通知邮件 |

### 2.3 第三方服务

| 服务 | 用途 |
|------|------|
| Vercel | 托管、边缘函数、环境变量 |
| Turso | 边缘 SQLite 数据库 |
| Resend |  transactional 邮件 |
| Vercel Analytics（可选） | 访问统计 |

## 四、前后端分离说明

本系统采用 **混合架构**：

- **静态内容**：履历、作品集、社交链接等以 JSON/Markdown 或 CMS 为数据源，构建时生成静态页面（SSG），首屏快、SEO 友好。
- **动态内容**：留言、联系表单、简历下载统计等通过 API Routes / Server Actions 与数据库交互。
- **前端渲染**：动画、3D、平滑滚动等纯前端逻辑在客户端 Hydration 后执行，不影响首屏 HTML。

**数据流**：
- 构建时：拉取作品集、履历配置 → 生成静态 HTML
- 运行时：留言/表单 → API → Turso；文件上传 → Blob；邮件 → Resend

## 五、安全与性能考量

### 5.1 安全

| 措施 | 说明 |
|------|------|
| 表单防刷 | 留言/联系表单使用 rate limit（如 Upstash Rate Limit） |
| 限流存储策略 | 若采用内存限流（非 Upstash），必须实现 **TTL 清理** 或 **LRU 上限**，防止 Map 无限增长导致 OOM；推荐优先使用 Upstash Rate Limit |
| 输入校验 | 服务端校验 + Zod 等 schema 校验 |
| 外部内容 URL 白名单 | content JSON（作品集、社交链接）中的 `url` 字段在渲染前必须校验协议，仅允许 `http:`、`https:`、相对路径 `/`，禁止 `javascript:` 等；建议在 `lib/utils/` 提供 `isValidHref(url)` 供组件调用 |
| 管理接口鉴权 | DELETE `/api/guestbook/[id]` 等管理接口上线前必须实现鉴权（Bearer Token 或 NextAuth），否则不得对外暴露 |
| 邮件失败处理 | 联系表单提交后，若 `sendContactNotificationEmail` 失败，应记录日志并考虑返回 `{ success: true, data: { mailSent: false } }` 或 503 |
| 文件上传 | 限制类型（PDF、图片）、大小、病毒扫描（可选） |
| 环境变量 | 数据库 URL、API Key 等仅存于 Vercel 环境变量 |
| CORS | API 仅允许同源或白名单域名 |

### 5.2 性能

| 措施 | 说明 |
|------|------|
| 静态优先 | 履历、作品集、社交页优先 SSG |
| 图片优化 | Next.js Image 组件，WebP/AVIF；使用 `next/image` 加载外部图片时，需在 `next.config.js` 的 `remotePatterns` 中配置允许的域名（如 `*.vercel-storage.com`、作品集 thumbnail 来源等） |
| 代码分割 | 3D/复杂组件动态 import，按需加载 |
| 边缘部署 | Vercel 全球 CDN + Turso 边缘数据库 |
| 缓存策略 | 静态资源长期缓存；API 响应可设短 TTL |

## 六、扩展与维护

- **外部图片**：`next.config.js` 的 `remotePatterns` 需配置作品集 thumbnail 等外部图源域名。
- **作品集管理**：初期可用 Markdown/JSON 配置；后期可接入 Sanity/Contentful 等 Headless CMS。
- **多语言**：如需 i18n，可引入 next-intl 或 next-i18next。
- **分析**：Vercel Analytics、Plausible 等隐私友好统计。

## 七、架构决策记录

| 决策 | 理由 |
|------|------|
| Next.js 而非纯 React | 需要 SSG、API、Server Actions 一体化，减少运维复杂度 |
| Turso 而非 Supabase | SQLite 边缘化更轻量，免费额度适合个人站点 |
| Framer Motion + 可选 Three.js | 兼顾开发效率与「视觉+交互酷」的拓展空间 |
| 混合架构 | 静态内容优先性能，动态功能按需接入，平衡体验与复杂度 |
